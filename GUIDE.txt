════════════════════════════════════════════════════════════════
  ЧАСТЬ 1 — ПОДГОТОВКА СЕРВЕРА
════════════════════════════════════════════════════════════════

┌─ ШАГ 1. Выбор и аренда VPS ───────────────────────────────────
│
│  Нужен VPS (виртуальный сервер) В ДРУГОЙ СТРАНЕ.
│  Минимальные требования:
│    • 1 CPU
│    • 512 MB RAM
│    • Ubuntu 22.04 LTS (рекомендуется)
│    • Любой объём диска (прокси почти не занимает место)
│
│  Где арендовать (дёшево и надёжно):
│    • Hetzner.com     — от 4€/мес, Германия/Финляндия
│    • DigitalOcean.com — от $6/мес, много локаций
│    • Vultr.com       — от $6/мес, много локаций
│    • Aeza.net        — от 150₽/мес, принимает крипту
│
│  После аренды у тебя будет:
│    IP адрес сервера (например: 123.45.67.89)
│    Пароль root пользователя
│
└────────────────────────────────────────────────────────────────

┌─ ШАГ 2. Подключение к серверу ────────────────────────────────
│
│  На Windows: скачай PuTTY (putty.org) или используй
│              встроенный терминал PowerShell / Windows Terminal
│
│  На Mac/Linux: открой Терминал
│
│  Команда подключения:
│
│    ssh root@ВАШ_IP
│
│  Пример:
│    ssh root@123.45.67.89
│
│  При первом подключении спросит "Are you sure?" — введи: yes
│  Потом введи пароль (символы не отображаются — это нормально)
│
└────────────────────────────────────────────────────────────────

┌─ ШАГ 3. Обновление системы ───────────────────────────────────
│
│  Выполни по одной команде, дожидайся завершения каждой:
│
│    apt update && apt upgrade -y
│
│  [Ждёшь пока обновится — может занять 1-2 минуты]
│
│    apt install -y curl wget git ufw python3 python3-pip unzip
│
│  [Ждёшь завершения]
│
└────────────────────────────────────────────────────────────────

┌─ ШАГ 4. Установка Docker ─────────────────────────────────────
│
│  Docker нужен для запуска прокси контейнера.
│
│    curl -fsSL https://get.docker.com | bash
│
│  [Ждёшь ~1-2 минуты]
│
│    systemctl enable docker && systemctl start docker
│
│  Проверяем что Docker работает:
│
│    docker --version
│
│  Должно вывести что-то вроде: Docker version 26.x.x
│  Если вывело — всё отлично, идём дальше.
│
└────────────────────────────────────────────────────────────────

┌─ ШАГ 5. Настройка файрвола ───────────────────────────────────
│
│  Открываем нужные порты:
│
│    ufw allow ssh
│
│    ufw allow 443/tcp
│
│    ufw --force enable
│
│  Проверяем:
│
│    ufw status
│
│  Должно показать:
│    443/tcp    ALLOW
│    22/tcp     ALLOW (или OpenSSH)
│
└────────────────────────────────────────────────────────────────


════════════════════════════════════════════════════════════════
  ЧАСТЬ 2 — ЗАГРУЗКА И УСТАНОВКА ПРОКСИ
════════════════════════════════════════════════════════════════

┌─ ШАГ 6. Загрузка файлов на сервер ────────────────────────────
│
│  У тебя есть архив mtproto_memes4u1337.zip
│
│  ВАРИАНТ А — с компьютера через scp (Mac/Linux):
│
│    scp mtproto_memes4u1337.zip root@ВАШ_IP:/root/
│
│  ВАРИАНТ Б — с компьютера через scp (Windows PowerShell):
│
│    scp mtproto_memes4u1337.zip root@ВАШ_IP:/root/
│
│  ВАРИАНТ В — прямо на сервере через wget (если выложил куда-то):
│
│    wget -O mtproto_memes4u1337.zip ССЫЛКА_НА_ФАЙЛ
│
└────────────────────────────────────────────────────────────────

┌─ ШАГ 7. Распаковка ───────────────────────────────────────────
│
│  На сервере (в SSH):
│
│    cd /root
│    unzip mtproto_memes4u1337.zip
│    cd mtproto_bot
│
│  Проверяем что все файлы на месте:
│
│    ls -la
│
│  Должны увидеть:
│    bot.py
│    mtproto_setup.sh
│    install_bot.sh
│    requirements.txt
│    .env
│    README.md
│
└────────────────────────────────────────────────────────────────

┌─ ШАГ 8. Запуск прокси ────────────────────────────────────────
│
│  Даём права на запуск:
│
│    chmod +x mtproto_setup.sh install_bot.sh
│
│  Запускаем установку прокси:
│
│    bash mtproto_setup.sh
│
│  ИЛИ с конкретным Fake TLS доменом (рекомендуется):
│
│    bash mtproto_setup.sh www.google.com
│
│  Лучшие домены для Fake TLS:
│    www.google.com    — работает везде (выбор по умолчанию)
│    cloudflare.com    — отличный DPI обход
│    www.microsoft.com — надёжно
│    apple.com         — хорошо в РФ/СНГ
│
│  [Скрипт работает ~30-60 секунд]
│
│  В конце скрипт выдаст ссылку вида:
│    tg://proxy?server=123.45.67.89&port=443&secret=ee...
│
│  ⚠️  СОХРАНИ ЭТУ ССЫЛКУ! Скопируй куда-нибудь.
│
│  Также ссылку всегда можно посмотреть командой:
│    cat /opt/mtproto_memes4u1337.conf
│
└────────────────────────────────────────────────────────────────

┌─ ШАГ 9. Проверка прокси ──────────────────────────────────────
│
│  Проверяем что контейнер запущен:
│
│    docker ps
│
│  Должен видеть строку с mtproto-memes4u1337 и статусом Up
│
│  Смотрим логи (должны быть без ошибок):
│
│    docker logs --tail 20 mtproto-memes4u1337
│
│  Если всё ок — переходим к боту.
│
└────────────────────────────────────────────────────────────────


════════════════════════════════════════════════════════════════
  ЧАСТЬ 3 — СОЗДАНИЕ И НАСТРОЙКА TELEGRAM БОТА
════════════════════════════════════════════════════════════════

┌─ ШАГ 10. Создание бота через BotFather ───────────────────────
│
│  1. Открой Telegram
│  2. Найди @BotFather и напиши ему
│  3. Отправь команду: /newbot
│  4. Введи название бота (любое, например: memes4u1337 proxy)
│  5. Введи username бота (должен заканчиваться на bot,
│     например: memes4u1337_proxy_bot)
│  6. BotFather пришлёт токен вида:
│       1234567890:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxx
│  7. СКОПИРУЙ ЭТОТ ТОКЕН — он нужен в следующем шаге
│
└────────────────────────────────────────────────────────────────

┌─ ШАГ 11. Узнай свой Telegram ID ──────────────────────────────
│
│  1. Найди в Telegram бота @userinfobot
│  2. Напиши ему /start или любое сообщение
│  3. Он ответит твоим ID — число вида 123456789
│  4. СКОПИРУЙ ЭТОТ ID
│
└────────────────────────────────────────────────────────────────

┌─ ШАГ 12. Заполнение конфига бота ─────────────────────────────
│
│  На сервере открой .env файл:
│
│    nano .env
│
│  Найди строки и замени значения:
│
│    BOT_TOKEN=ВАШ_ТОКЕН_СЮДА
│      ↓ меняем на ↓
│    BOT_TOKEN=1234567890:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxx
│
│    ADMIN_ID=0
│      ↓ меняем на ↓
│    ADMIN_ID=123456789
│
│  Сохраняем файл:
│    Нажми Ctrl+X
│    Нажми Y
│    Нажми Enter
│
└────────────────────────────────────────────────────────────────

┌─ ШАГ 13. Установка и запуск бота ─────────────────────────────
│
│  Запускаем скрипт установки:
│
│    bash install_bot.sh
│
│  [Скрипт установит зависимости и создаст системный сервис]
│
│  Запускаем бота:
│
│    systemctl start mtproto-bot
│
│  Проверяем статус:
│
│    systemctl status mtproto-bot
│
│  Должно показать: Active: active (running)
│
│  Если всё ок — бот пришлёт тебе сообщение в Telegram!
│
└────────────────────────────────────────────────────────────────


════════════════════════════════════════════════════════════════
  ЧАСТЬ 4 — ИСПОЛЬЗОВАНИЕ
════════════════════════════════════════════════════════════════

┌─ Команды в боте ───────────────────────────────────────────────
│
│  /start    — Главное меню с кнопками
│  /status   — Статус контейнера и аптайм
│  /link     — Ссылка для подключения к прокси
│  /logs     — Последние логи контейнера
│  /restart  — Перезапуск прокси
│  /regen    — Новый секрет (все текущие подключения сбросятся)
│
│  Кнопки в меню делают то же самое — удобнее на телефоне.
│
└────────────────────────────────────────────────────────────────

┌─ Как раздать прокси друзьям ───────────────────────────────────
│
│  Вариант 1 — поделись ссылкой:
│    Скопируй ссылку tg://proxy?server=...
│    Отправь другу — он нажмёт и Telegram сам добавит прокси
│
│  Вариант 2 — поделись ботом:
│    Дай другу username твоего бота
│    Он напишет /start и получит ссылку автоматически
│
│  Вариант 3 — вручную:
│    Telegram → Настройки → Конфиденциальность → Тип соединения
│    → Использовать прокси → Добавить прокси → MTProto
│    Введи: Сервер, Порт, Секрет из файла /opt/mtproto_memes4u1337.conf
│
└────────────────────────────────────────────────────────────────

┌─ Watchdog (автомониторинг) ────────────────────────────────────
│
│  Бот сам проверяет прокси каждые 5 минут.
│  Если прокси упал — пришлёт тебе алерт с кнопкой Рестарт.
│  Если поднялся после падения — тоже уведомит.
│
│  Изменить интервал проверки:
│    nano /opt/mtproto_bot/.env
│    Измени строку: CHECK_INTERVAL=180  (в секундах)
│    systemctl restart mtproto-bot
│
└────────────────────────────────────────────────────────────────


════════════════════════════════════════════════════════════════
  ЧАСТЬ 5 — ПОЛЕЗНЫЕ КОМАНДЫ НА СЕРВЕРЕ
════════════════════════════════════════════════════════════════

  Посмотреть ссылку прокси:
    cat /opt/mtproto_memes4u1337.conf

  Логи прокси (в реальном времени):
    docker logs -f mtproto-memes4u1337

  Логи бота (в реальном времени):
    journalctl -u mtproto-bot -f

  Перезапустить прокси:
    docker restart mtproto-memes4u1337

  Перезапустить бота:
    systemctl restart mtproto-bot

  Остановить всё:
    docker stop mtproto-memes4u1337
    systemctl stop mtproto-bot

  Пересоздать прокси с новым секретом:
    bash /opt/mtproto_setup.sh

  Пересоздать прокси с другим доменом:
    bash /opt/mtproto_setup.sh cloudflare.com

  Статус всех контейнеров:
    docker ps

  Статус бота:
    systemctl status mtproto-bot


════════════════════════════════════════════════════════════════
  ЧАСТЬ 6 — РЕШЕНИЕ ПРОБЛЕМ
════════════════════════════════════════════════════════════════

┌─ Прокси не подключается ───────────────────────────────────────
│
│  1. Проверь что контейнер запущен:
│       docker ps
│
│  2. Проверь что порт открыт в файрволе:
│       ufw status
│     Должен быть 443/tcp ALLOW
│
│  3. Проверь логи прокси на ошибки:
│       docker logs --tail 30 mtproto-memes4u1337
│
│  4. Попробуй другой Fake TLS домен:
│       bash /opt/mtproto_setup.sh cloudflare.com
│
└────────────────────────────────────────────────────────────────

┌─ Бот не отвечает ──────────────────────────────────────────────
│
│  1. Проверь статус:
│       systemctl status mtproto-bot
│
│  2. Проверь логи:
│       journalctl -u mtproto-bot -n 50
│
│  3. Убедись что токен и ID правильные:
│       cat /opt/mtproto_bot/.env
│
│  4. Перезапусти:
│       systemctl restart mtproto-bot
│
└────────────────────────────────────────────────────────────────

┌─ Контейнер не запускается ─────────────────────────────────────
│
│  1. Смотри логи:
│       docker logs mtproto-memes4u1337
│
│  2. Удали и пересоздай:
│       docker stop mtproto-memes4u1337
│       docker rm mtproto-memes4u1337
│       bash /opt/mtproto_setup.sh
│
└────────────────────────────────────────────────────────────────

┌─ Порт 443 занят ───────────────────────────────────────────────
│
│  Скрипт автоматически найдёт другой порт (8443, 2053 и т.д.)
│  Если хочешь освободить 443 вручную:
│
│    ss -tuln | grep :443      ← узнать что занимает порт
│    systemctl stop nginx      ← если это nginx
│    systemctl stop apache2    ← если это apache
│
└────────────────────────────────────────────────────────────────
